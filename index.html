方法論核心機制

1. 術語感知切分 (Term-Aware Chunking)

依法規天然結構（條→款→目）切分，非固定 token 數。每個 chunk 保留術語的完整定義���下文， 定義段落標記為「不可切割」。平均 chunk 長度 811 字元，每 chunk 平均包含 3.2 個術語。

2. ���層術語詞典

  • 層級 A — 法定明確定義術語（48 個）：逐字抄錄，不得改寫
  • 層級 B — 隱含核心概念（14 個）：從多條文歸納，AI 初稿 + SME 審核
  • 層級 C — 附錄具體實例（37 個）：AI 批次生成

3. 混合檢索 + RRF 融合

三路檢索同時執行，以 Reciprocal Rank Fusion (k=60) 合併結果：

  • Vector Search (權重 0.3) — 語意相似度，text-embedding-3-large
  • Keyword Search (權重 0.3) — 精確字詞匹配，相當於 BM25
  • Term Metadata Match (權重 0.4) — 術語 ID 精確匹配 + 定義 chunk 加權 1.5x

4. 動態術語注入 + Token 預算管理

根據查詢和檢索結果，動態選擇相關術語注入 System Prompt：

  • Direct Hit — 查詢中直接出現的術語 → Full 格式（定義 + 差異 + 禁用詞）
  • Indirect — 檢索結果中出現的術語 → Medium 格式（定義前 150 字）
  • Background — 知識圖譜 1-hop 擴展 → Short 格式（60 字摘要）

總預算 6,000 tokens，按優先級依序分��直到耗盡。

5. Synonym Map 查詢改寫

從術語詞典的 prohibitedAlternatives 自動建立對照表，在查詢階段攔截禁用替代詞： 

「網路銀行」 → 「電子銀行」
「線上銀行」 → 「電子銀行」
「數位銀行」 → 「電子銀行」
「手機銀行」 → 「行動銀行」
「密碼」     → 依情境區分「固定密碼」/「一次性密碼」
